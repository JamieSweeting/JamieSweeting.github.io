Car Controller

With realistic engine simulation

Overview

As part of a university module, I created a realistic car controller, applying real world physics to better inform how the car would drive.

The current design takes into account torque, RPM and gear ratios to simulate an engine receiving throttle and increasing power. It uses Unity's built in wheel collider system which, while not always easy to work with, allowed my focus to be on the engine performance.

Timeline

Using my experience with version control, I set up a Github repo and began planning the next steps ahead. I used Agile methodology to help manage my time, following a process of:

  1. Planning what mechanic to add next and how I would attempt to go about it (i.e. ideating on slip detection for an automatic counter steering system).

  2. Implementing the mechanic, usually starting with a primitive solution and iterating on it until it accomplishes what I set out.

  3. Testing and refining the mechanic, ensuring it runs as intended under different circumstances and gathering feedback from anyone who can give second opinions before moving onto the next task.
With the help of online resources, I slowly implemented different aspects of the engine, making sure to adhere to realism whenever possible.
